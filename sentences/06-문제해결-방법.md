# 06. 문제 해결·방법

17개 핵심 개념어를 문장 기반으로 학습한다.

---

## 문장 01

> 문제 해결의 첫 단계는 문제에 대한 접근 방식을 정하는 것이며, 어떤 전략을 선택하느냐에 따라 이후의 절차와 결과가 크게 달라진다.

### 문장 분해
- 주장: 문제 해결의 첫 단계 = 접근 방식 결정
- 결론: 전략 선택이 절차와 결과를 좌우한다

### 핵심 개념어

| 개념어 | 이 문장에서의 의미 |
|--------|---------------------|
| 해결 | 문제 상황을 분석하고 원하는 상태에 도달하는 행위 |
| 접근 | 문제를 다루기 위해 취하는 방향이나 관점 |
| 전략 | 목표 달성을 위해 수립하는 전체적인 계획 |
| 절차 | 목표에 도달하기 위해 순서대로 밟는 단계들 |

### 구조 요약
문제 해결 과정을 접근→전략→절차의 순서로 연결하며, 전략의 중요성을 강조하는 구조

### 포함 범주
`문제 해결·방법`

## 문장 02

> 이론을 현실에 적용할 때는 상황에 맞는 방법을 선택해야 하며, 하나의 방법이 모든 상황에 유효하지는 않으므로 항상 대안을 함께 마련해 두어야 한다.

### 문장 분해
- 조건: 이론을 현실에 적용할 때
- 주장 A: 상황에 맞는 방법 선택이 필요하다
- 주장 B: 하나의 방법이 만능이 아니므로 대안이 필요하다

### 핵심 개념어

| 개념어 | 이 문장에서의 의미 |
|--------|---------------------|
| 적용 | 이론이나 원리를 구체적 상황에 실제로 사용하는 것 |
| 방법 | 목적을 달성하기 위해 사용하는 구체적 수단이나 방식 |
| 선택 | 여러 가능성 중에서 하나를 골라 결정하는 행위 |
| 대안 | 기존 방법이 실패하거나 부적합할 때 대신 채택할 수 있는 다른 방안 |

### 구조 요약
적용 시 방법 선택의 필요성을 제시한 뒤, 대안 마련의 중요성으로 확장하는 구조

### 포함 범주
`문제 해결·방법`

## 문장 03

> 시행착오는 다양한 시도와 실패를 반복하며 점차 올바른 해법에 접근하는 방식인 반면, 발견적(heuristic) 방법은 경험에서 얻은 직관적 규칙을 활용하여 빠르게 근사적 해답을 찾는 방식이다.

### 문장 분해
- 정의 A: 시행착오 = 시도와 실패 반복으로 해법에 접근
- 정의 B: 발견적 방법 = 경험 기반 직관적 규칙으로 빠르게 근사 해답 도출
- 관계: 대조

### 핵심 개념어

| 개념어 | 이 문장에서의 의미 |
|--------|---------------------|
| 시행착오 | 성공할 때까지 다양한 시도와 실패를 반복하는 문제 해결 방식 |
| 발견적(heuristic) | 완벽하지 않지만 경험적 규칙을 이용하여 빠르게 답에 도달하는 방식 |

### 구조 요약
시행착오와 발견적 방법을 속도와 정확성의 차이를 기준으로 대조하는 구조

### 포함 범주
`문제 해결·방법`

## 문장 04

> 최적화란 주어진 제약 조건 안에서 목표를 가장 효과적으로 달성하는 방안을 찾는 과정이며, 제약 조건이 바뀌면 최적의 해답도 함께 달라진다.

### 문장 분해
- 정의: 최적화 = 제약 조건 내에서 가장 효과적인 방안을 찾는 과정
- 주장: 제약 조건 변화 → 최적 해답도 변화

### 핵심 개념어

| 개념어 | 이 문장에서의 의미 |
|--------|---------------------|
| 최적화 | 주어진 조건 하에서 최선의 결과를 도출하는 과정 |
| 제약 조건 | 문제 해결 과정에서 반드시 충족해야 하는 한계나 제한 사항 |

### 구조 요약
최적화를 제약 조건과의 관계로 정의한 뒤, 조건 변화에 따른 해답의 가변성을 설명하는 구조

### 포함 범주
`문제 해결·방법`

## 문장 05

> 효율성을 높이려면 투입 대비 산출을 극대화해야 하지만, 효율성과 안전성 사이에는 흔히 트레이드오프가 존재하여 한쪽을 높이면 다른 쪽이 낮아지는 관계가 발생한다.

### 문장 분해
- 정의: 효율성 = 투입 대비 산출의 극대화
- 주장: 효율성과 안전성 사이에 트레이드오프가 존재한다
- 결론: 한쪽을 높이면 다른 쪽이 낮아진다

### 핵심 개념어

| 개념어 | 이 문장에서의 의미 |
|--------|---------------------|
| 효율성 | 투입한 자원 대비 얻어지는 성과의 비율 |
| 트레이드오프 | 하나를 얻기 위해 다른 하나를 포기해야 하는 상충 관계 |

### 구조 요약
효율성을 정의한 뒤, 트레이드오프 개념을 통해 효율성 추구의 한계를 제시하는 구조

### 포함 범주
`문제 해결·방법`

## 문장 06

> 피드백이란 행위의 결과가 다시 원인 측에 되돌아가 이후의 행위를 조절하는 메커니즘이며, 시스템이 스스로를 교정하고 개선하는 데 핵심적 역할을 한다.

### 문장 분해
- 정의: 피드백 = 결과가 원인 측으로 되돌아가 이후 행위를 조절하는 메커니즘
- 결론: 시스템의 자기 교정과 개선에 핵심적이다

### 핵심 개념어

| 개념어 | 이 문장에서의 의미 |
|--------|---------------------|
| 피드백 | 결과 정보가 원인 측으로 되돌아가 다음 행위를 조절하는 순환 과정 |

### 구조 요약
피드백의 메커니즘을 정의한 뒤, 시스템 개선에서의 기능적 역할을 설명하는 구조

## 문장 07

> 모델링이란 복잡한 현실을 단순화하여 핵심 변수와 관계만 추출한 모형을 만드는 작업이며, 시뮬레이션은 그 모형을 가상으로 작동시켜 다양한 조건에서의 결과를 예측하는 과정이다.

### 문장 분해
- 정의 A: 모델링 = 현실을 단순화하여 핵심 변수와 관계를 추출한 모형 제작
- 정의 B: 시뮬레이션 = 모형을 가상 작동시켜 다양한 조건의 결과를 예측
- 관계: 모델링 → 시뮬레이션 (순차)

### 핵심 개념어

| 개념어 | 이 문장에서의 의미 |
|--------|---------------------|
| 모델링 | 복잡한 현실에서 핵심 요소만 추출하여 단순화된 모형을 구성하는 작업 |
| 시뮬레이션 | 모형을 가상 환경에서 실행하여 다양한 시나리오의 결과를 미리 확인하는 과정 |

### 구조 요약
모델링과 시뮬레이션을 순차적 관계로 정의하며, 각각의 기능을 구분하는 구조

### 포함 범주
`문제 해결·방법`
